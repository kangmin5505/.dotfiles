`$ARGUMENTS`로 전달된 주제에 대해 Jira 코멘트를 **한국어**로 작성해줘.

**입력:**
- 인자가 있으면 해당 주제에 대해 코드베이스를 분석하여 작성할 것.
- 인자가 없으면 사용자에게 주제를 질문할 것.

**작성 원칙:**
- 주제에 대해 **간단 명료하게** 핵심만 작성할 것. 장황한 서술 금지.
- 코드베이스를 분석하여 기술적 근거를 포함할 것. 추측하지 마.
- 부족한 정보는 `[TODO: ...]` 플레이스홀더를 넣어줘.

**출력 형식 — Jira wiki markup:**
- **결과물은 반드시 `/tmp/jira-comment.txt` 파일로 저장할 것.** 채팅에는 파일 경로만 안내하고 본문을 직접 출력하지 마.
- **인라인 코드 금지**: Jira wiki의 `{{코드}}` 문법을 절대 사용하지 마. 중괄호가 Jira 매크로 파서와 충돌하여 깨진다. 코드/식별자는 그냥 일반 텍스트로 쓸 것.
- 형식: 볼드 제목 + 넘버링 섹션 + 불렛 포인트의 간결한 구조.
- 구조 예시:

{noformat}
*볼드 제목*

1. 섹션 제목
* *키워드:* 설명 텍스트
* *키워드:* 설명 텍스트

2. 섹션 제목
* 불렛 내용
* 불렛 내용
{noformat}
